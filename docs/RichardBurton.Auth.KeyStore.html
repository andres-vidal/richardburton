    RichardBurton.Auth.KeyStore (richard_burton v0.1.0)

    Caches a provider's JWKS signing keys and keeps them fresh so that key rotation does not require restarting the server.

    Provider-agnostic: it caches whatever any RichardBurton.Auth.JWKS implementation returns. The implementation is configured via the :jwks_provider application env and can be overridden per instance with the :provider option.

    Keys are refreshed:

    • on a schedule, honouring the max-age advertised by the provider; and
    • lazily, whenever a token presents a kid that is not currently cached (rate-limited by :refresh_cooldown to avoid hammering the provider with bogus kids).
