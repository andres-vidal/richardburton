    RichardBurton.Auth.Session (richard_burton v0.1.0)

    Server-side (stateful) app sessions. Each session is a row in auth_sessions holding the authenticated user's subject_id and the SHA-256 hash of an opaque token. The raw token is set as the rb-session cookie and looked up on every request, so a session can be revoked instantly by deleting its row.

    Lifetime is a sliding idle timeout (:session_idle_timeout) refreshed on use, bounded by an absolute cap (:session_max_age) measured from creation.
