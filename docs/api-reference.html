    API Reference richard_burton v#0.1.0

    Modules

    RichardBurton

    RichardBurton keeps the contexts that define your domain and business logic.

    RichardBurton.Auth

    Authentication/authorization boundary. Delegates to the implementation configured under :auth_service (default RichardBurton.Auth.Google, swapped for a Mox mock in tests).

    RichardBurton.Auth.Claims

    Validates the claims of an OIDC ID token: issuer, audience, subject and expiry. Signature verification is performed separately (see RichardBurton.Auth.Google); this module only checks the decoded claims.

    RichardBurton.Auth.Csrf

    Double-submit CSRF token: a signed, session-bound token issued as a readable (http_only: false) cookie at login and echoed back in the rb-csrf-token header on state-changing admin requests (see RichardBurtonWeb.Plugs.VerifyCsrf).

    RichardBurton.Auth.Google

    Google identity-provider implementation of the RichardBurton.Auth behaviour.

    RichardBurton.Auth.JWKS

    Behaviour for fetching a JSON Web Key Set (JWKS): the signing keys and issuer used to verify ID tokens, plus the cache lifetime the provider advertises.

    RichardBurton.Auth.JWKS.Google

    RichardBurton.Auth.JWKS implementation that fetches from Google: the issuer from Google's OpenID configuration and the signing keys from its OAuth2 certs endpoint (respecting the certs' Cache-Control: max-age).

    RichardBurton.Auth.JWKS.Stub

    RichardBurton.Auth.JWKS implementation that fetches nothing.

    RichardBurton.Auth.KeyStore

    Caches a provider's JWKS signing keys and keeps them fresh so that key rotation does not require restarting the server.

    RichardBurton.Auth.Recaptcha

    Behaviour for recaptcha authorization services

    RichardBurton.Auth.Recaptcha.Google

    Google Recaptcha implementation for RichardBurton.Auth.Recaptcha behaviour

    RichardBurton.Auth.Session

    Server-side (stateful) app sessions. Each session is a row in auth_sessions holding the authenticated user's subject_id and the SHA-256 hash of an opaque token. The raw token is set as the rb-session cookie and looked up on every request, so a session can be revoked instantly by deleting its row.

    RichardBurton.Reference

    Schema for publication references: free-text provenance entries that record how a publication's data was collected, so a reader can verify the claim. Owned by a single publication (deleted with it), ordered, and edited inline via the publication update path — never shared or deduplicated across publications.

    RichardBurtonWeb

    The entrypoint for defining your web interface, such as controllers, views, channels and so on.

    RichardBurtonWeb.AuthorController
    RichardBurtonWeb.DevSessionController

    Dev/test-only credentials provider. Mints an admin rb-session (plus the readable csrf-token) without the Google OAuth handshake, so the admin UI can be exercised locally.

    RichardBurtonWeb.EmailController
    RichardBurtonWeb.Endpoint

    Conveniences for translating and building error messages.

    RichardBurtonWeb.ErrorView
    RichardBurtonWeb.Plugs.Authenticate.Bearer

    Authenticates a request via an identity-provider token in the Authorization: Bearer header (see RichardBurton.Auth) and assigns :subject_id. Used by the login endpoints that exchange the provider token for a session.

    RichardBurtonWeb.Plugs.Authenticate.Cookie

    Authenticates a request via the app's own rb-session cookie (see RichardBurton.Auth.Session) and assigns :subject_id.

    RichardBurtonWeb.Plugs.Authorize.Admin

    Authorizes the request only if the authenticated subject (assigned by an Authenticate plug) has the :admin role.

    RichardBurtonWeb.Plugs.Authorize.Recaptcha

    Plug for recaptcha verification

    RichardBurtonWeb.Plugs.VerifyCsrf

    Double-submit CSRF check for state-changing admin requests. Safe methods (GET/HEAD/OPTIONS) pass through untouched. For unsafe methods it requires an rb-csrf-token header carrying a valid RichardBurton.Auth.Csrf token whose subject matches the authenticated session (assigned by Authenticate.Cookie, which must run first). Defense in depth on top of the SameSite=Lax cookie.
