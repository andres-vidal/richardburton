# Publication store

The state layer behind every publication component, built on a single
[Jotai](https://jotai.org/) store (`modules/store`). Components never hold
publication state locally — they read derived atoms through hooks and mutate via
imperative actions, so the same data drives the list, the table, the editable
review, and the counters in sync.

## Shape

- **Base atoms** — the raw state: the id list, per-row families
  (`publicationFamily`, `overrideFamily`, `errorFamily`, `deletedFamily`, …),
  and UI flags (validating, focused row, attribute visibility).
- **Derived atoms** — computed views: `visibleIds` (non-deleted), `validIds`
  (no error), the various counts, and a merged `visiblePublicationFamily` that
  layers pending edits over the stored value.
- **Actions** — imperative writers that operate on the module store directly:
  `setAll`, `overrideField`, `addNew`, `duplicate`, `setDeleted`,
  `focusNextInvalid`, the `reset*` family, and so on.
- **Hooks** — thin `useAtomValue` wrappers (`useVisiblePublicationIds`,
  `useIsPublicationValid`, `usePublicationField`, …) that components subscribe to.

Because actions write to the same store the hooks read from, a seed or an edit
anywhere is reflected everywhere. This is exactly what the stories rely on: the
shared fixtures call actions (`resetAll`, `setAll`) and the rendered components
react.
